The problem

Iron abundance is the field's loudest systematic.

Reflection fits routinely return iron abundances scattered from sub-solar to ten times solar — for the same sources. The spread is driven not by astrophysics alone but by choices: model variant, coronal geometry, disk density, and instrument.

When a parameter that should be roughly universal varies by an order of magnitude across the literature, that scatter is the result worth studying. But it can only be studied population-wide if the values are gathered consistently.

Why a database, not a review table. In the spirit of García et al. (2018) Fig. 3, Kalpha records every published AFe with its full context — variant, instrument, whether the value was free, fixed, or pegged at the model boundary — so any subset can be re-queried and re-plotted. The plot below is generated live from that catalogue, not redrawn by hand.
Beyond iron. The same machinery catalogues reflection-based black-hole spin measurements with a quality-tiered curation scheme, building a census for population and systematics studies.
The centerpiece

XRB vs AGN iron abundance, live from Kalpha

Free fits include values the optimizer drove to the model boundary (shown as pegged/limit); fixed fits are frozen at a chosen value, off by default. The statistics recompute for whichever sample the toggles select — exact per-sample values from the database, not binned estimates. Watch the AGN median collapse to solar when fixed values are shown: hundreds of AGN fits freeze AFe at 1× solar.

How it's built

From PDF to queryable catalogue

A systematic, PRISMA-style pipeline turns refereed reflection papers into one normalized table, then serves it through a programmable interface.

01

Discover

Refereed RELXILL-family papers identified through NASA ADS.

02

Extract

AI-assisted parsing of each PDF — AFe, spin, variant, instrument, fit flags — with the exact table/column recorded as provenance.

03

Curate

Expert review and quality tiering; bibcode and value corrections logged, not silently overwritten.

04

Merge

Per-paper JSON consolidated into literature.csv / .json.

05

Serve

Exposed as an MCP server with query, statistics, and plotting tools.

The tools

search_literature

Filter fits by source, class, model variant, instrument, year, and AFe range.

compute_literature_statistics

Median, mean, quartiles and counts over any filtered subset.

generate_iron_histogram

The AFe distribution that powers the plot above — faceted by class, instrument, or variant.

semantic_search

Meaning-based retrieval across analyzed papers, via Voyage AI embeddings.

find_fe_abundance_patterns

Heuristic AFe extraction from a paper's text during ingestion.

+ schema in progress

Electron density (log ne) and per-fit QC tiers are being folded into the schema.

Use it

Query Kalpha yourself

Kalpha speaks the Model Context Protocol. Add it as a connector in a compatible client and ask questions in plain language, or drive the tools directly through the API.

MCP endpoint

https://kalpha-production.up.railway.app/mcp

This site refreshes its plot from that same server. The included kalpha_afe.py regenerates afe_data.json; a scheduled GitHub Action can keep it current.

# Refresh the catalogue snapshot behind the plot
pip install "mcp>=1.0"
python kalpha_afe.py            # writes afe_data.json
python kalpha_afe.py --serve    # view the live plot locally
python kalpha_afe.py --png      # export a publication figure

# Example questions once connected in an MCP client:
#   "median A_Fe for free NuSTAR fits of AGN"
#   "histogram of A_Fe for MAXI J1820+070"
#   "all relxillCp spin measurements above 0.9"
